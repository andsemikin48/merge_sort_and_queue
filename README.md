# Сравнительный анализ алгоритмов сортировки: Слиянием vs Пузырьковая

В данном проекте проводится экспериментальное исследование времени выполнения двух алгоритмов сортировки,
реализованных на языке Python: алгоритма слиянием (Merge Sort) и алгоритма пузырьком (Bubble Sort).
Цель работы — проанализировать зависимость времени работы алгоритмов от размера входных данных
и подтвердить их теоретическую сложность.

## Полученные результаты

### 1. Сортировка слиянием (Merge Sort)
Алгоритм характеризуется гарантированной сложностью **O(n log n)**. Экспериментальные данные это подтверждают:
при увеличении размера массива время растет предсказуемо медленно.

| Размер массива (n) | Время выполнения (мс) |
|--------------------|-----------------------|
| 10                 | 0.021                 |
| 100                | 0.194                 |
| 1000               | 2.511                 |
| 20 000             | 70.707                |
| 200 000            | 877.442               |

### 2. Пузырьковая сортировка (Bubble Sort)
Алгоритм имеет квадратичную сложность **O(n²)**. На малых данных он показывает приемлемые результаты,
однако при увеличении объема данных время выполнения неприемлемо возрастает.

| Размер массива (n) | Время выполнения      |
|--------------------|-----------------------|
| 20                 | 0.035 мс              |
| 200                | 2.397 мс              |
| 2 000              | 168.600 мс            |
| 20 000             | 15.297 с  (15 297 мс) |
| 200 000            | ~1560.84 с (~26 минут)|


## Сравнительный анализ

### График зависимости времени от размера входа
(Визуально можно представить как резко растущую кривую для пузырьковой сортировки и пологую для сортировки слиянием)

- **На малых данных (до 1000 элементов):**
    - Оба алгоритма работают быстро (доли миллисекунд).
    - Пузырьковая на 20 элементах оказалась даже быстрее слияния (0.035/2 ~0.017 мс против 0.021 мс на 10 элементах),
      что объясняется низкими накладными расходами на рекурсию и выделение памяти в простейшем алгоритме.
- **На средних данных (2000 элементов):**
    - Merge Sort уверенно лидирует, тратя всего ~2.5 мс на 1000 эл.
    - Bubble Sort начинает существенно отставать (168 мс на 2000 эл.), что в ~67 раз медленнее.
- **На больших данных (20000 элементов):**
    - Merge Sort справляется за 70 миллисекунд.
    - Bubble Sort требует уже 15 секунд.
- **На очень больших данных (200000 элементов):**
    - Merge Sort укладывается в 877 миллисекунд (менее секунды).
    - Bubble Sort показывает неприемлемый результат: **1560 секунд (более 26 минут)**.

## Выводы

Сортировка слиянием (Merge Sort) демонстрирует высокую эффективность 
на всех диапазонах данных. Время выполнения растет предсказуемо медленно благодаря 
логарифмическому фактору в сложности O(n log n).
